# 测试案例 - AI 合伙分钱方案生成器 V0

## 案例 1：一人出钱，一人全职

| 字段 | 值 |
|------|-----|
| 合伙人数 | 2 |
| 合伙人 A | 出资 20 万，仅出资，提供启动资金 |
| 合伙人 B | 出资 5 万，全职运营，负责选品、运营、客服 |
| 预期年利润 | 30-50 万 |
| 口头约定 | 暂定五五分 |
| 联系方式 | test_wechat_001 |

测试重点：
- 资金与出力权重分配是否合理
- 合伙人 B 全职运营是否有合理回报
- AI 是否能识别"仅出资"和"全职运营"的差异

## 案例 2：三人合伙

| 字段 | 值 |
|------|-----|
| 合伙人数 | 3 |
| 合伙人 A | 出资 30 万，仅出资，提供启动资金 |
| 合伙人 B | 出资 5 万，全职运营，负责日常运营和管理 |
| 合伙人 C | 出资 1 万，供应链，提供货源资源和供应链管理 |
| 预期年利润 | 50-100 万 |
| 口头约定 | 无 |
| 联系方式 | test_wechat_002 |

测试重点：
- 三人分配逻辑是否自洽
- 资源型合伙人（C）的权重是否合理
- 无口头约定时的建议

## 案例 3：双方都出钱，一方全职

| 字段 | 值 |
|------|-----|
| 合伙人数 | 2 |
| 合伙人 A | 出资 10 万，全职运营，负责选品、运营、客服 |
| 合伙人 B | 出资 10 万，兼职供应链，负责供应链和物流对接 |
| 预期年利润 | 10-20 万 |
| 口头约定 | 无 |
| 联系方式 | test_wechat_003 |

测试重点：
- 双方出资相等但出力不同时的分配
- 兼职与全职的权重差异
- 利润较小时的分配方案

## 测试步骤

### 技术验收

1. 启动服务：`npm start`
2. 打开浏览器访问 `http://localhost:3000`
3. 在浏览器控制台输入 `fillTestCase(1)` 填充案例1
4. 点击提交，确认：
   - 数据保存到 SQLite
   - AI 返回报告
   - 前端渲染 Markdown
5. 重复案例2和案例3
6. 验证 API：`curl http://localhost:3000/api/cases`

### 业务验收

1. 提交 3 个案例并确认完整跑通
2. 邀请 3-5 个真实目标用户试用
3. 确认至少 1 个用户愿意付费
4. 人工审核单份报告控制在 15-30 分钟
