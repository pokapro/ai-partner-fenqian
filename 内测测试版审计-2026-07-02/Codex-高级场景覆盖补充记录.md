# Codex 高级场景覆盖补充记录

> 时间：2026-07-02  
> 执行人：Codex  
> 目标：把已知缺口从“未覆盖提醒”升级为“高级补充模块”，用于测试版内测。

## 一、为什么补

用户指出：既然已经知道常见缺口，就应该直接补充。

本次处理原则：

- 基础合伙场景继续作为 V0 主线。
- 高级复杂场景不作为强承诺卖点，但系统必须能识别并输出结构化建议。
- 税务、婚姻继承、融资法务等高风险内容只输出框架和风险边界，不给确定法律/税务结论。

## 二、已补充的高级场景

| 场景 | 关键词 | 输出要求 |
| --- | --- | --- |
| 员工期权 / Vesting | 期权、期权池、员工持股、ESOP、RSU、vesting、成熟、cliff | 期权池比例、成熟安排、离职处理、回购/失效规则 |
| 融资条款 | 融资、投资人、天使、A轮、对赌、反稀释、优先清算 | 接受/调整/拒绝三套方案、触发公式、创始人底线 |
| 治理结构 | 董事会、董事席位、GP、LP、持股平台、AB股、一票否决 | 股东会/董事会/经营层权责、席位、否决权、控制权风险 |
| 税务与分红合规 | 个税、分红税、核定征收、税务筹划 | 分红/工资/劳务/服务费边界、会计师复核提示 |
| 婚姻继承 | 婚姻、离婚、夫妻财产、身故、继承、遗产 | 收益权与股东身份区分、继承人进入限制、优先回购 |
| IP / 用工 | 专利、商标、软著、代码、账号、劳动合同、社保、全职、兼职 | IP归属、账号资产、全职投入、兼职成果、劳动关系边界 |

## 三、代码调整

### `server/prompt_dt.js`

- 增加 `L3+ 高级补充模块`。
- 普通报告输出新增 `## 📜 L1+ 高级补充模块`。
- 用户主动提到高级关键词时，必须输出：
  - 适用前提
  - 方案对比表
  - 可写入协议的条款草案
  - 风险边界
  - 待补充信息

### `server/prompt_dt_protocol.js`

- 协议包输出新增 `## 六、高级事项附件`。
- 用户要求生成协议时，高级事项必须进入协议附件，不能只写风险提示。

### `server/framework_gaps.js`

- 将语义从“未覆盖缺口”调整为“高级触发清单”。
- 高级关键词仍会记录，用于内测后判断哪些模块需要升级为正式决策树节点。

## 四、当前结论

补充后，系统不再只是覆盖基础 70%-80% 场景，而是：

- 基础场景：可作为 V0 主功能稳定输出。
- 高级场景：可作为补充模块输出，支持内测和案例沉淀。
- 高风险场景：输出边界明确，建议人工/律师/税务师复核。
