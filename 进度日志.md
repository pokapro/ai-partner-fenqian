## 2026-06-05 (Day 1 - 数据库历史案例参考能力)

### 完成状况：✅ 完成

### 改动目标
AI 不再只根据表单内容生成报告，而是：
1. 从数据库读取历史相似案例
2. 结合当前案例 + 历史案例数据给出更准确的分配建议
3. 保证输出质量，不会跑偏

### server/db.js
- [x] 新增 `findSimilarCases(partners, limit=5, excludeId)` 方法
  - 匹配逻辑：根据出资模式（等额/不等额/0元出资）和出力类型组合进行相似度评分
  - 评分维度：合伙人数(+30)、出资模式(+30)、出力类型精确匹配(+20)、部分重叠(+10)、出资金额量级(+20)
  - 按相似度 + 时间排序，返回脱敏案例列表
- [x] 新增 `getCaseStats()` 方法
  - 返回：总案例数、有报告数、各分配方案采纳率、审核状态分布、付款意向比例、出资模式分布
- [x] 脱敏处理：只保留 partnerCount, fundingMode, effortTypes, totalCapital, allocationScheme
- [x] 新增辅助函数：`classifyCase()` 案例分类、`similarityScore()` 相似度评分、`sanitizeCase()` 脱敏、`extractAllocationScheme()` 提取分配方案

### server/prompt.js
- [x] 系统 Prompt 增加「历史案例参考」指引
- [x] AI 需要知道哪些方案在类似案例中更常用，但必须声明"历史数据仅供参考"
- [x] 新增 `buildReferenceContext(similarCases, stats)` 函数
  - 格式化输出平台统计 + 相似案例列表
  - 带 ⚠️ 警示声明
- [x] 导出新函数供 ai.js 使用

### server/ai.js
- [x] `generateReport` 函数增加 `dbRef` 参数
- [x] 调用 AI 前先查数据库获取相似案例 + 统计
- [x] 相似案例数据作为 system message 传给 AI
- [x] 所有 real provider（ollama/glm/deepseek/qwen）统一通过 `buildMessages()` 构建包含参考上下文的 messages
- [x] mock provider 也支持 referenceContext 参数
- [x] 数据库查询失败不影响主流程（非致命错误）
