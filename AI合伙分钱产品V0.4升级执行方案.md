# AI 合伙分钱产品 V0.4 升级执行方案

## 1. 升级目标

当前 V0.3 已完成结构化工作台、AI 报告生成、CopilotKit 顾问、知识库匹配和付款意向记录。

V0.4 的目标是把产品从“AI 生成合伙分钱报告”升级为：

> 合伙关系诊断 + 分钱方案 + 五权结构 + 协议条款包

核心升级方向：

- 引入成熟股东合作协议样本，形成专家知识壁垒。
- 补齐所有权、分红权、经营权、决策权、退出权诊断。
- 报告从普通建议升级为可用于合伙人沟通的决策文件。
- 明确 29.9 元和 99 元付费档位差异。
- 形成明显优于豆包、通义、ChatGPT 普通问答的结构化能力。

## 2. 产品定位调整

### 当前定位

AI 合伙分钱诊断工具。

### 升级后定位

AI 合伙关系诊断与分钱方案工具。

产品不只回答“怎么分钱”，而是帮助用户判断：

- 谁出钱？
- 谁出力？
- 谁分红？
- 谁经营？
- 谁拍板？
- 谁监督？
- 谁退出？
- 谁违约怎么办？

## 4. 新增知识案例

新增 1 条 `knowledge_cases`。

```text
id: kc_mature_shareholder_agreement_4p
title: 四人股东合作协议：代持、表决权、任职考核与退出机制综合设计
partner_count: 4
scene_type: 多股东公司治理与分钱机制
funding_pattern: 不等额出资
effort_pattern: 任职股东 + 非任职股东 + 代持股东
recommended_scheme: 治理型方案
source: 成熟协议样本脱敏
status: active
```

核心矛盾：

```text
多名股东共同设立公司，股权登记、实际权益、经营管理、重大事项表决、任职股东考核、退出和违约处理均需要提前约定，否则容易出现控制权冲突、分红争议、代持争议和股东僵局。
```

推荐方案：

```text
股权比例、分红权、经营权、表决权、监督权和退出权分开设计。重大事项采用 67% 以上表决，一般事项采用 50% 以上表决；任职股东接受岗位绩效考核，非任职股东只参与股权分红；涉及代持的股东需另行签署代持协议。
```

## 5. 新增规则库

至少新增 8 条 `rules`。

### 5.1 股权代持风险规则

触发条件：

```text
代持、名义股东、实际股东、股权登记人与实际权益人不一致
```

建议：

```text
存在股权代持时，必须单独签署代持协议，明确实际出资人、名义持有人、分红权归属、表决权行使方式、退出方式、违约责任和争议处理方式。主股东协议中只做代持关系提示，不应替代完整代持协议。
```

风险等级：high  
优先级：90

### 5.2 重大事项 67% 表决规则

触发条件：

```text
三人以上股东、股权分散、无绝对控股人、重大事项决策冲突
```

建议：

```text
三人以上且股权较分散时，重大事项建议采用 67% 以上表决门槛，包括增资减资、融资担保、利润分配、修改章程、合并分立、清算解散、重大投资和核心人员任免。一般事项可采用 50% 以上表决。
```

风险等级：high  
优先级：85

### 5.3 任职股东与非任职股东区分规则

触发条件：

```text
股东中有人在公司任职，有人只出资不参与经营
```

建议：

```text
应区分任职股东和非任职股东。任职股东可领取岗位工资、绩效奖金和股权分红，但必须接受岗位绩效考核；非任职股东不参与日常经营收益，只按照股权或协议约定参与分红。
```

风险等级：medium  
优先级：85

### 5.4 任职股东绩效不达标处理规则

触发条件：

```text
任职股东、岗位职责、绩效考核、连续不达标
```

建议：

```text
任职股东连续未达到基础绩效目标时，应触发股东会评议机制，可约定降薪、调岗、减少经营权限、稀释激励份额或退出任职角色。涉及股权调整的，应设置明确表决门槛和书面程序。
```

风险等级：medium  
优先级：75

### 5.5 红黄绿线行为约束规则

触发条件：

```text
股东行为规范、损害公司利益、挪用资金、泄露秘密、私自经营竞品
```

建议：

```text
多人合伙建议设置红线、黄线、绿线行为机制。红线行为包括挪用资金、泄露商业秘密、收受回扣、私自经营竞品、严重损害公司利益等，可触发赔偿和强制退出；黄线行为给予一次整改机会；绿线行为纳入日常奖惩。
```

风险等级：high  
优先级：90

### 5.6 退出分类规则

触发条件：

```text
退股、退出、合伙人不干了、违约退出、死亡、离婚、无民事行为能力
```

建议：

```text
退出机制应分为主动退出、被动退出和法定退出。主动退出需提前书面申请并完成清算；被动退出适用于未出资、严重违约、损害公司利益等情形；法定退出适用于死亡、离婚、无民事行为能力等特殊情况。
```

风险等级：high  
优先级：90

### 5.7 竞业限制与客户资源保护规则

触发条件：

```text
退出后从事同类业务、客户资源、供应链资源、竞业限制
```

建议：

```text
核心股东或任职股东退出后，建议约定合理期限内不得直接或间接从事与公司相同或相近业务，不得带走客户、供应商、员工和商业秘密。竞业限制条款应注意合理范围和可执行性。
```

风险等级：medium  
优先级：70

### 5.8 公章与财务监督规则

触发条件：

```text
公司已成立、公章、财务账户、对账、账簿、监督
```

建议：

```text
多人合伙公司应明确公章保管、用印审批、财务账簿查阅、定期对账和财务报告机制。建议建立用印台账，重大合同或担保事项需经股东会授权。
```

风险等级：high  
优先级：80

## 7. 合同输出格式要求

成熟协议样本的价值不只在条款内容，还包括合同版式、章节顺序、表达习惯和签署结构。V0.4 需要把“合同格式要求”一并收录为模板标准。

### 7.1 输出文件类型

产品后续输出协议类内容时，应优先支持以下文件形态：

- 股东合作协议书
- 合伙分钱约定书
- 股权代持协议提示清单
- 股东行为规范
- 退出机制补充协议
- 重大事项决策规则
- 任职股东绩效约定

V0.4 阶段不要求生成正式法律合同，但报告中的“协议条款草稿”必须接近正式合同的结构和表达。

### 7.2 合同标题格式

标题应居中，使用明确文件名：

```text
股东合作协议书
```

或根据场景生成：

```text
合伙分钱与股东合作约定书
```

```text
合伙人退出机制补充协议
```

标题下方不得使用营销化文案。

### 7.3 当事人信息格式

合同开头应列明各方主体信息，格式参考：

```text
甲方（股东）：__________
身份证号码：__________
身份证地址：__________

乙方（股东）：__________
身份证号码：__________
身份证地址：__________

丙方（股东）：__________
身份证号码：__________
身份证地址：__________
```

如为公司主体，可扩展：

```text
公司名称：__________
统一社会信用代码：__________
注册地址：__________
法定代表人：__________
```

注意：

- 产品输出时默认使用空白占位符，不输出用户真实身份证信息。
- 用户输入的真实姓名可用于报告展示，但协议草稿中建议使用甲方、乙方、丙方代称。
- 涉及真实身份信息的正式合同应由用户自行补充或由律师确认。

### 7.4 前言/订立背景格式

合同正文前应有一段订立背景，说明协议目的、适用范围和与公司章程的关系。

推荐格式：

```text
依据《中华人民共和国公司法》等相关法律规定，各方因共同投资、经营项目/公司形成合作关系。为进一步明确各方在公司设立、运营管理、股权结构、利润分配、决策机制、退出机制、行为规范等事项中的权利义务，经各方协商一致，订立本协议。

本协议在不违反法律法规及公司章程强制性规定的前提下，对各方具有约束力。涉及工商登记、股权变更、合同效力等事项的，建议另行咨询专业律师或相关机构。
```

### 7.5 一级章节顺序

股东合作协议类文件建议采用以下章节顺序：

```text
一、公司/项目基本信息
二、股东出资方式及股权结构
三、股东实缴和出资证明
四、股东权利与义务
五、利润分配与亏损承担
六、公司组织架构及岗位职责
七、议事规则与重大事项表决
八、任职股东考核机制
九、股东行为规范与红线机制
十、股权转让
十一、退出机制
十二、竞业限制与保密义务
十三、增资入股及新项目投资
十四、协议解除、终止与清算
十五、其他约定
十六、签署页
```

如果用户仍处于“未注册公司、仅准备合伙”的阶段，可简化为：

```text
一、合作项目基本信息
二、各方投入与职责
三、利润分配方式
四、亏损承担方式
五、经营管理与决策规则
六、账目公开与财务管理
七、退出与违约处理
八、保密与竞业限制
九、争议处理
十、签署页
```

### 7.6 条款编号格式

推荐格式：

```text
一、章节标题
1、一级条款内容
（1）二级条款内容
a、三级列举内容
```

要求：

- 同一份协议内编号风格保持一致。
- 章节标题使用中文数字：一、二、三。
- 条款使用阿拉伯数字：1、2、3。
- 细分项使用中文括号：（1）（2）（3）。
- 禁止在同一协议中混用过多编号样式。

### 7.7 股权结构表达格式

涉及股权和出资时，建议统一表达：

```text
甲方以现金/实物/知识产权/资源等方式出资，出资额为人民币____万元，占公司注册资本/项目权益的____%，甲方持有____%的股权/分红权。
```

如存在代持，应增加：

```text
如上述股权存在名义持有人与实际权益人不一致的情形，各方应另行签署《股权代持协议》，明确实际出资人、名义持有人、权益归属、表决权行使、分红收取、退出和违约责任等事项。
```

### 7.8 决策权表达格式

重大事项建议使用明确表决比例，不使用“大家商量”“协商决定”等模糊表达。

推荐格式：

```text
涉及增资减资、融资担保、利润分配、修改公司章程、合并分立、解散清算、重大投资、核心管理人员任免等重大事项，须经代表三分之二以上表决权的股东同意后方可执行。

除上述重大事项外，一般经营事项经代表二分之一以上表决权的股东同意后执行。
```

### 7.9 任职股东与非任职股东表达格式

如股东中有人参与经营，有人不参与经营，应明确区分：

```text
在公司内部担任具体岗位或承担经营职责的股东，为任职股东。任职股东应按照公司岗位职责和绩效要求参与考核，并可根据岗位获得工资、福利和绩效奖金。

不在公司内部担任具体岗位、不参与日常经营管理的股东，为非任职股东。非任职股东原则上不领取岗位工资或绩效奖金，仅按照本协议约定享有股东分红。
```

### 7.10 红线行为表达格式

股东行为约束应采用“禁止行为 + 后果”的结构。

推荐格式：

```text
股东、董事、监事、高级管理人员不得实施以下行为：
（1）挪用公司资金；
（2）私设账户或将公司资金以个人名义存储；
（3）未经股东会同意，以公司资产为他人提供担保；
（4）泄露公司商业秘密；
（5）利用职务便利谋取属于公司的商业机会；
（6）私自经营与公司相同或相近业务；
（7）收受回扣、佣金或其他不当利益；
（8）其他严重损害公司或其他股东利益的行为。

发生上述行为并造成公司或其他股东损失的，违约方应承担赔偿责任；情节严重的，可按本协议约定启动强制退出程序。
```

### 7.11 退出机制表达格式

退出机制必须分类表达，不能只写“可以退股”。

建议分为：

- 主动退出
- 被动退出
- 法定退出

推荐格式：

```text
股东主动退出的，应提前____日向公司及其他股东提交书面申请，并在完成财务清算、债务清偿、资料交接和股权转让手续后退出。

股东存在未履行出资义务、严重违反本协议、损害公司利益、泄露商业秘密、私自经营竞品等情形的，经约定比例表决权同意后，可启动被动退出程序。

股东死亡、离婚、丧失民事行为能力或发生其他法定事由的，各方应根据法律规定及本协议约定处理其股权或权益继承、转让、回购等事项。
```

### 7.12 竞业限制表达格式

竞业限制应避免过度绝对化，建议提示“合理期限、合理范围”。

推荐格式：

```text
核心股东或任职股东退出后，在合理期限和合理范围内，不得直接或间接从事与公司主营业务相同或相近的业务，不得带走公司客户、供应商、员工、商业秘密或其他核心资源。具体期限、地域范围、业务范围及补偿安排，建议由各方另行协商并经专业人士确认。
```

### 7.13 签署页格式

协议末尾必须保留签署区。

推荐格式：

```text
甲方（签字/盖章）：__________
身份证号码：__________
日期：______年____月____日

乙方（签字/盖章）：__________
身份证号码：__________
日期：______年____月____日

丙方（签字/盖章）：__________
身份证号码：__________
日期：______年____月____日
```

如为公司主体：

```text
公司（盖章）：__________
法定代表人/授权代表：__________
日期：______年____月____日
```

### 7.14 协议输出风格要求

协议草稿输出必须满足：

- 表达正式、克制，不使用营销化语言。
- 不使用“AI 建议你”“根据模型判断”等表述。
- 不泄露知识库、案例库、模板库来源。
- 不承诺法律效力。
- 不直接生成用户真实身份证等敏感信息。
- 保留空白占位符，便于用户或律师补充。
- 关键条款必须有明确触发条件、执行方式和后果。
