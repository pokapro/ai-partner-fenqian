### 3.2 hotfix2 — 后台登录修复

**时间**：2026-06-15 下午  
**Commit**: `e982936` (tag `v0.7.2-hotfix2`)

**问题**：Admin HTML 内联 `<script>` 中用 `const`/`let` 声明函数，导致 `function` 不是全局变量，`onclick` 属性绑定的函数名找不到。

**根因链路**：
```
const/let 在顶层作用域
  → 阻止同一 <script> 中的 function 成为全局变量
  → 按钮 onclick="adminAction()" 找 window.adminAction
  → window.adminAction 是 undefined
  → 点击按钮无反应（静默失效）
```

**修复**：所有 `const`/`let` 函数声明改为 `var` 或直接 `function xxx()` 声明。

**验证方法**：
1. 打开 Admin 页面，打开 Console
2. 输入 `typeof adminAction` → 原来是 `undefined`，修复后应为 `function`

**部署状态**：✅ 已在 Render 上线

### 3.5 hotfix5 — fetch 超时 + AbortError 友好提示

**时间**：2026-06-15 傍晚  
**Commit**: `03b334e`

**问题**：Render 免费实例冷启动耗时最长 50 秒。如果用户在冷启动窗口提交生成，fetch 会超时抛出 `TypeError: fetch failed` → catch 到后显示"网络错误，请检查连接后重试"——误导用户以为是自己的网络问题。

**修复内容**：
1. **AbortController 65s 超时**：所有 DeepSeek API 调用添加 `AbortController`，超时 65 秒
2. **错误分类提示**：
   - `AbortError` → "请求超时，服务器正在启动，请稍后重试"
   - 其他网络错误 → "网络错误，请检查连接后重试"
3. **影响文件**：`server/ai.js` + `server/copilotkit.js` 的 fetch 调用

**相关代码片段**：
```javascript
const controller = new AbortController();
const timeoutId = setTimeout(() => controller.abort(), 65000);

try {
  const response = await fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json', 'Authorization': `Bearer ${apiKey}` },
    body: JSON.stringify({ ... }),
    signal: controller.signal
  });
  clearTimeout(timeoutId);
  // ...
} catch (error) {
  clearTimeout(timeoutId);
  if (error.name === 'AbortError') {
    throw new Error('请求超时，服务器正在启动，请稍后重试');
  }
  throw new Error('网络错误，请检查连接后重试');
}
```

### 3.6 hotfix6 — marked v18 表格渲染崩溃修复

**时间**：2026-06-15 18:02  
**Commit**: `6065fc8` (tag `v0.7.2-hotfix6`)

**⭐ 最关键的修复 — 解决了生成报告后空白页问题**

**问题**：`src/ChatApp.jsx` 中的 `fixTables(html)` 函数用正则表达式对 marked 输出的 HTML 做表格格式美化。但 **marked v18（CDN 版本）** 的 table renderer 传递的 `header`/`body` 参数是 **tokens 对象**而非字符串（与 v17 不同），导致 `fixTables` 内部直接调用 `.replace()` 时报 `TypeError: e.replace is not a function`，React 渲染崩溃 → 白屏。

**修复**：在 `renderer.table` 入口加兼容层

```javascript
renderer.table = function(header, body) {
  // ★ 兼容层：marked v18 传的是 tokens 对象，不是字符串
  // header = { tokens: [...] } 或 string 或 array
  if (typeof header !== 'string') {
    if (header && header.tokens) {
      header = this.parser.render(header);
    } else if (Array.isArray(header)) {
      header = header.map(t => {
        if (typeof t === 'string') return t;
        if (t && t.tokens) return this.parser.render(t);
        return t?.text || '';
      }).join('');
    } else if (typeof header?.text === 'string') {
      header = header.text;
    } else {
      header = String(header || '');
    }
  }
  // body 同理...
  
  // 兼容层后继续原来的 .replace() 逻辑
  return originalFixTablesLogic(header, body);
};
```

**三种格式兼容**：
| 格式 | 示例 | 处理方式 |
|------|------|---------|
| string | `"<tr>...</tr>"` | 直接使用（v17 行为） |
| tokens object | `{ tokens: [...] }` | 用 `parser.render()` 还原 |
| array | `[{ tokens: [...] }, ...]` | 遍历每个元素 render 后拼接 |
| object with text | `{ text: "..." }` | 取 `.text` |
| 其他 | `null/undefined/number` | `String(x || '')` |

## 6. 持久化问题根因分析

### 6.1 核心发现

> **Render Dashboard → Disk 页面显示：**
> **"Disks are not supported for free instance types."**

Render **免费实例不支持持久化 Disk**。SQLite 数据库文件 `data/app.db`（通过 `~/.fenqian-data/app.db` 路径存储）每次部署重建都会丢失。

### 6.2 影响范围

| 受影响 | 不受影响 |
|--------|---------|
| 已生成的案例数据（cases 表） | 代码（Git 版本管理） |
| 订单记录（orders 表） | static 资源（包含在构建产物中） |
| 知识库种子数据（seed.js 幂等插入，部署后会重新生成） | 前端交互逻辑 |
| 付款申请记录 | AI 模型配置（env vars） |

### 6.3 解决方案选项

| 方案 | 优点 | 缺点 | 成本 |
|------|------|------|------|
| **A. Render 付费实例** (Starter $7/月) | 最小改动，支持 Disk 持久化 | 需要绑定信用卡 | $7/月 |
| **B. SQLite 存 GitHub** (CI/CD 持久化) | 零成本 | 并发写冲突、仓库膨胀 | 免费 |
| **C. MongoDB Atlas 免费版** (512MB) | 真正的远程持久化 | 需改代码、引入依赖 | 免费 |
| **D. 腾讯云 VPS 迁移** | 完全控制、可挂载持久卷 | 需要手动迁移、管理 | ~¥30/月 |

**当前建议**：短期（Free tier），方案 C（MongoDB Atlas）或凌晨自动备份到 GitHub；中期（有付费用户后），方案 A（升级 Render Starter $7/月）最省事。

## 8. 关键教训

### 8.1 技术教训

| # | 教训 | 来源 |
|---|------|------|
| 1 | **const/let 阻止 function 全局注册**：`const fn = () => {}` 和 `function fn() {}` 在 script 作用域内的全局可见性不同。HTML `onclick="fn()"` 绑定的是 `window.fn`，而 `const fn` 不在 window 上。结论：inline script 中用 `var` 或 `window.fn = function` | hotfix2 |
| 2 | **marked v18 不兼容 v17 table renderer 参数格式**：v17 传 string，v18 传 tokens object。CDN 加载的版本不可控，必须加兼容层。结论：升级 marked 版本时必须验证 table 渲染 | hotfix6 |
| 3 | **Render 免费实例无 Disk 持久化**：免费计划的文档明确标注。结论：使用 Render Free 时数据必须存在外部服务 | 持久化分析 |
| 4 | **DEEPSEEK_API_KEY 读取策略不一致**：同一个 key 在不同文件读法不同。结论：建立统一的 key 读取函数 | hotfix3/hotfix4 |
| 5 | **Render 冷启动 50s**：免费实例 spin down 后的首次请求极慢。结论：所有 fetch 必须设超时 + 友好提示 | hotfix5 |

### 8.2 流程教训

| # | 教训 | 来源 |
|---|------|------|
| 1 | **hotfix 按严重度排序部署**：后台登录(P0) → 报告生成(P0) → AI 顾问(P0) → 超时优化(P1) → 配置修复(P2)。优先保证主流程可用 | 全链 |
| 2 | **部署失败不等同于代码错误**：hotfix6 首次部署失败（Exited status 1）是构建环境问题，重试后成功。失败后先看 deploy logs 确认错误类型 | hotfix6 |
| 3 | **API 版本号 ≠ 实际功能版本**：package.json 的 `version` 字段没改时健康检查返回旧版本号，但前端 JS hash 能准确反映实际更新 | 验证 |
| 4 | **"形成 skill" 严格按字面意思**：不自行扩大范围，保持单一职责 | 交叉经验 |

### 8.3 诊断流程标准化（从本次事件提炼）

```
发现故障
  ↓
1. 确认当前线上版本（健康检查 + JS hash + Render Events）
2. 复现故障（浏览器操作 + 观察错误）
3. 缩小范围（后端 API curl 测试 vs 前端渲染观察）
4. 定位根因（Console 错误 + 源码分析 + CDN 版本差异）
5. 本地修复（改代码 + node -c / npm build 验证）
6. Git commit + push
7. 等待 Render 自动部署（或手动触发）
8. 部署后验证（浏览器复测 + API 检查）
9. 记录到进度日志 + 记忆层同步
```
